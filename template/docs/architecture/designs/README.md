# Design docs

One file per component or cross-cutting concern, grouped by domain:
`designs/<domain>/<component>.md` (e.g. `designs/auth/session-handling.md`).

These are **living** documents — they describe the system as it is *now* and are
updated in the same PR as the code they describe. History and rationale belong in
ADRs, not here. Create one with the `/vibe-design` skill, which starts from
`../_templates/design.md`.
